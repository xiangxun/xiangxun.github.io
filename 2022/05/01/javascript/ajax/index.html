前端笔记 AJAX

AJAX简介

AJAX 全称Asynchronous JavaScript And XML，异步JS和XML
通过AJAX可以在浏览器中向服务器发送异步请求，最大的优势：页面无刷新获取数据
AJAX不是新的编程语言，而是一种将现有的标准组合结合在一起使用的方法

XML简介
XML 可拓展标记语言
XML 被设计用来传输和存储数据
XML和HTML类似，不同的是HTML中都是预定义标签，而XML中没有预定义标签，全都是自定义标签，用来表示一些数据

AJAX优点

  • 可以无需刷新页面与服务器进行通信
  • 允许你根据用户事件来更新部分页面内容

AJAX缺点

  • 没有浏览历史，不能回退
  • 存在跨域问题
  • 不利于SEO优化

AJAX的使用

核心对象：XMLHttpRequest
AJAX的所有操作都是通过该对象进行的

使用步骤
1、创建xhr实例对象
const xhr = new XMLHttpRequest();
绑定监听
2、指定发送请求的：method url
xhr.open(“GET”, “http://localhost:8000/test_get”);
3、发送请求
xhr.send();

 0：未初始化（Uninitialized）。尚未调用 open()方法。
 1：已打开（Open）。已调用 open()方法，尚未调用 send()方法。
 2：已发送（Sent）。已调用 send()方法，尚未收到响应。
 3：接收中（Receiving）。已经收到部分响应。
 4：完成（Complete）。已经收到所有响应，可以使用了。

method URL 参数
能带什么参数:

query  ?name="zhangsan"&age=18  urlencoded

params /zhangsan/26

跨域

  1. 什么是跨域
    浏览器为了安全，而采用的同源策略 Same origin policy
  2. 同源策略
    同源策略是由Netscape提出的一个著名的安全策略，现在所有支持JavaScript的浏览器都会使用这个策略
    web是构建在同源策略基础之上的，浏览器只是针对同源策略的一种实现
    所谓同源是指：协议、域名、端口都相同，才能算是在同一个源里
  3. 非同源受到哪些限制
    Cookie不能读取
    DOM无法获得
    AJAX请求不能获取数据

JSONP

JSONP是什么
JSONP JSON with Padding 是一个非官方的跨域解决方案，只支持GET请求
JSONP怎么工作
绕过xhr，使用script、img、link、iframe等标签发送请求不受同源策略的限制

CORS

CORS (Cross-Origin Resource Sharing)，跨域资源共享，CORS是官方的跨域解决方案，它的特点是不需要在客户端做任何特殊的操作，完全在服务器中进行处理，支持所有常见的请求

app.get(‘’,(request,response)=>{
response.setHeader(‘Access-Control-Allow-Origin’,‘url’)
//url为可以允许访问的URL 。 * 通配符 ，允许所有的地址访问
response.setHeader(‘Access-Control-Expose-Headers’,‘*’)
})

PUT 、DELECT请求属于复杂请求
请求之前需要有嗅探请求

const cors = require(‘cors’)
app.use(cors())
